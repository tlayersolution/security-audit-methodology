## üõ†Ô∏è Audit Methodology (The 4-Phase Cycle)

| Phase | Description | Tools Used |
| :--- | :--- | :--- |
| **01. Reconnaissance** | Subdomain discovery, tech stack fingerprinting, and attack surface mapping. | Nmap, Subfinder, Whois |
| **02. Dynamic Analysis (DAST)** | Real-time testing of active APIs and web interfaces for injections and bypasses. | Burp Suite Professional, OWASP ZAP |
| **03. Static Analysis (SAST)** | Deep-dive code review of **Java/Spring Boot** and **Node.js** codebases for logic flaws. | SonarQube, Manual Code Review |
| **04. Remediation** | Delivery of a technical report with specific code patches and hardening guides. | Git, Docker (Container Security) |

## üîç Key Assessment Areas

### üîë Authentication & Identity
- **JWT Hardening:** Testing for weak signatures, algorithm confusion, and lack of expiration.
- **OAuth/Auth0 Integrations:** Ensuring secure token exchange and scope validation.

### üîå API & Backend Security
- **Injection Prevention:** Mitigation of SQLi, NoSQLi, and Command Injections in **PostgreSQL** and **MongoDB** environments.
- **Rate Limiting:** Protecting resources against Brute Force and DoS attacks.
- **Broken Object Level Authorization (BOLA):** Ensuring users only access their own data.

### üê≥ Infrastructure & Cloud
- **Docker Security:** Scanning for vulnerable base images and misconfigured container privileges.
- **Deployment Hardening:** Auditing **Vercel** and **Render** configurations for secure headers and SSL integrity.
